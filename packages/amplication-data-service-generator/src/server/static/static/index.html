Following is an example for how to use your application. It shows how to fetch a list of users. In the sample code below, replace the placeholder URL [your-app-url], with the real address of your application. Learn more in our docs.

Sample Code

JavaScript

var myHeaders = new Headers();
myHeaders.append("Content-Type", "application/json");
myHeaders.append("Authorization", "Basic YWRtaW46YWRtaW4=");

var requestOptions = {
  method: 'GET',
  headers: myHeaders,
  redirect: 'follow'
};

fetch("https://[your-app-url]/users", requestOptions)
  .then(response => response.text())
  .then(result => console.log(result))
  .catch(error => console.log('error', error));

cURL

curl --location --request GET 'https://[your-app-url]/users' \
--header 'Content-Type: application/json' \
--header 'Authorization: Basic YWRtaW46YWRtaW4='

HTTP Request

GET /users HTTP/1.1
Host: [your-app-url]
Content-Type: application/json
Authorization: Basic YWRtaW46YWRtaW4=

API URL structure

You can access any of the entities in your application by using a URL that follows this structure: 
    [method] https://[your-app-url]/[entity-name]/[optional-id] 
      
{JSON body}
  • [method] - supports GET/ POST/ DELETE/
  • [your-app-url] - the URL of your application
  • [entity-name] - the name of the entity
  • [optional-id] - the ID of a specific record

Example

GET https://sampleapp.amplication.app/customer/12345
POST https://sampleapp.amplication.app/customer

Authentication

Please note that all endpoints are secured with HTTP basic authentication.
By default, your app comes with one user with the username "admin" and password "admin".
In the example above the authorization header uses the default username and password.
To learn how to use basic authentication, visit https://swagger.io/docs/specification/authentication/basic-authentication/

Authorization

All endpoints use role-based authorization, based on the access permissions that you set per entity in amplication.
